# GREEN-API test UI

Одностраничное приложение для вызова методов GREEN-API:

- `getSettings`
- `getStateInstance`
- `sendMessage`
- `sendFileByUrl`

## Стек

- Vite (vanilla TypeScript)
- TypeScript
- axios
- json-formatter-js

## Локальный запуск

```bash
npm install
npm run dev -- --host 0.0.0.0 --port 5173
```

По умолчанию приложение будет доступно по адресу `http://localhost:5173`.

## Как пользоваться страницей

1. Создайте инстанс в личном кабинете GREEN-API и авторизуйте телефон (сканируйте QR-код).
2. На странице в блоке **Подключение к GREEN-API**:
   - введите `idInstance`;
   - введите `ApiTokenInstance`;
   - нажмите **Сохранить подключение**.
3. Используйте блок **Методы GREEN-API**:
   - **getSettings** — нажмите кнопку, чтобы получить текущие настройки инстанса;
   - **getStateInstance** — нажмите кнопку, чтобы получить состояние инстанса;
   - **sendMessage**:
     - `chatId` — номер в формате `79991234567@c.us` или id группы `XXXXXXXXXX-XXXXXXXXXX@g.us`;
     - `Текст сообщения` — текст до 20000 символов;
     - нажмите **Отправить сообщение**;
   - **sendFileByUrl**:
     - `chatId` — как выше;
     - `urlFile` — ссылка на файл, начинающаяся с `http://` или `https://`;
     - `fileName` — имя файла с расширением (например, `image.png`);
     - `caption` — опциональная подпись к файлу;
     - нажмите **Отправить файл**.

## Ответы и ошибки

- Все ответы методов отображаются в нижнем блоке:
  - короткий статус над JSON (успех/ошибка);
  - подробный форматированный JSON-ответ/ошибка с помощью `json-formatter-js`.
- Сообщения об ошибках учитывают коды HTTP:
  - `400` — проблемы с параметрами запроса (Validation failed и т.п.);
  - `401` — проблемы с `ApiTokenInstance`;
  - `403` — проблемы с `idInstance` или URL;
  - `429`, `500`, `502` — подсказки по повтору запроса или снижению частоты.
